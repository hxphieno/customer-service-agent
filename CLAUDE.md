## Project Overview

**竞赛**：[具有多模态能力的客服智能体设计](https://www.datafountain.cn/competitions/1165)（DataFountain，奖金 ¥145,000）

任务：基于产品手册知识库，构建多模态客服智能体，对 400 道用户问题给出高质量回答（含文本+配图）。

完整竞赛原文已保存至 `competition_details.md`。

---

## 核心任务要求

智能体需同时具备：
1. **多模态理解**：理解用户的文字问题（含附图）
2. **多模态 RAG**：检索产品手册文本 + 手册插图，回答中引用相关图片
3. **多轮对话 + 幻觉抑制**：通过思维链拆解多问题，逐一作答，避免无依据编造
4. **（可选）微调**：基于多模态对话数据微调大模型

初赛还需提交：RESTful API（`/chat` 端点）、源码、技术文档、验证报告。

---

## 评分机制

**初赛总分 = 系统设计（30%）+ 技术实现（70%）**

技术实现由 **LLM 自动评分**，每题 1-5 分：
| 分值 | 标准 |
|------|------|
| 1 | 未回应问题，结构混乱，图片无关 |
| 2 | 部分回应，图文结合差 |
| 3 | 回应问题但缺乏深度，图片有一定帮助 |
| 4 | 清晰全面，结构合理，图片有助理解 |
| 5 | 详细深度，图文完美互补，显著提升理解 |

最终得分 = 所有题总分 / 满分，返回 [0.0, 1.0]。采用 **AB 榜**：A 榜持续开放，B 榜 2026-06-20 开放。

## 数据格式

### `question_public.csv`
- 列：`id`, `question`（共 ~416 题）
- 多行 CSV，单题可能跨多行（带引号）
- 两类问题：①通用电商客服问题（退换货、物流等）②需查手册的产品技术问题

### `手册/*.txt`（产品手册）
每个 `.txt` 文件是**单行 JSON 数组**，必须 `json.loads()` 解析：
```python
[text_content, [image_id_list]]
# text_content: 手册正文，<PIC> 标记图片位置
# image_id_list: 按顺序对应每个 <PIC> 的图片 ID
```
图片文件在 `手册/插图/<image_id>.png`。

### `submission_example.csv`（提交格式）
- 列：`id`, `ret`
- `ret` 的**正确格式**（参考官方样例）：
  ```
  "回答文字 <PIC> 更多文字 <PIC>", ["image_id_1", "image_id_2"]
  ```
  即：文本中用 `<PIC>` 标记图片插入位置，同时附带对应图片 ID 列表。
- 不需要图片时：直接输出回答文本字符串。
- **注意**：ID 不连续，提交必须精确匹配 `question_public.csv` 中的 id。

### `手册/汇总英文手册.txt`
所有手册的英文版本合并文件（可用于英文模型的检索）。
