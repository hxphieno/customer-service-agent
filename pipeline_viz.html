🔧 Agent Pipeline - 完整技术架构

Phase 1.5 详细实现 | retry_count 管理 | 无限循环防护

📊 重试次数管理
重试策略:
retry_count=0 → 1st 生成 (无 feedback)
feedback set → retry_count=1
retry_count=1 → 2nd 生成 (with feedback)
feedback set again → retry_count=2
retry_count=2 → ❌ END (不再生成)

判断条件: if retry_count < 1
🔄 路由决策树
route_after_validation()
执行顺序很重要!
if validation_passed → LanguageTranslator
elif is_merged → END ❌
elif search_failures OR partial_retrieval → Filler
elif retry_count < 1 → AnswerGenerator
else → END ❌ (forced_pass)
🛡️ 防无限循环机制
1. is_merged 标记
SubQuestionFiller 返回时:
is_merged=True
一旦设置，后续失败直接 END

2. retry_count 上限
AnswerGenerator 增加:
if feedback: retry_count += 1
route 检查: retry_count < 1

3. search_failures 保持
进入 Filler 后不会清除
避免重复判断进入 Filler
📤 节点输入输出
QuestionAnalyzer
输入: question, image_b64
输出: question_type, product, sub_questions, sub_q_dependent
约束: sub_questions ≤ 3
Retriever
输入: sub_questions, product
输出: retrieved_chunks, search_failures, product_verified, partial_retrieval
关键: search_failures 只在此设置
AnswerGenerator
输入: retrieved_chunks, validation_feedback, retry_count
输出: draft_answer, used_images, [retry_count+1]
仅有 feedback 时增加 retry_count
Validator
输入: draft_answer, used_images, sub_questions
输出: validation_passed, [validation_feedback], [final_answer]
final_answer 仅在通过时设置
SubQuestionFiller
输入: search_failures, validation_feedback
输出: filler_answers, is_merged=True
必须返回 is_merged=True!
AnswerMerger
输入: draft_answer, filler_answers
输出: draft_answer, used_images, is_merged=True
不修改 retry_count
LanguageTranslator
输入: final_answer, question
输出: final_answer (翻译后)
若 final_answer="" 返回空
⚠️ 诊断信息类型
diagnose_coverage_gap():
检索失败: search_failures 非空
→ 无法补救，进不了 Filler

拆分问题: partial_retrieval=true
→ 可以进 Filler 补充全部 sub_questions

生成问题: 其他情况
→ 可以 retry_count < 1 时重新生成
🔀 Filler 触发条件
✅ 进入 Filler:
validation_passed=False AND
(search_failures 非空 OR partial_retrieval=true)

❌ 不进 Filler:
诊断为"检索失败"且
无法推断其他失败原因
🚫 强制通过防线
MAX_VALIDATOR_RETRIES=2
备用机制:
if retry_count ≥ 2
→ Validator 直接返回
validation_passed=True

不依赖此机制，应由路由逻辑优先阻止
